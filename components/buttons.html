Buttons

Use buttons to trigger an immediate action such as saving a document. They are used in forms, toolbars, dialog footers and as stand-alone action triggers.

Default button

The default button applies to most use cases. When you add the default button to your interface, it appears enabled and ready to use.

<button class="tuiButton tst-button-default">Default</button>

Modifiers

Primary
To call attention to an action on a form, or highlight the strongest call to action on a page. Primary buttons should only appear once per screen (not including the application header or in a modal dialog). Keep in mind that not every screen needs a primary button.
Subtle
Use when a toolbar or default buttons would draw attention away from more important content. It is often used at the top of a page or section and must use an icon and label.
Caution
Only use this button type to call increased attention to an action that can have irreversible consequences.
<button class="tuiButton tuiButton--primary">Primary</button>
<button class="tuiButton tuiButton--subtle">Subtle</button>
<button class="tuiButton tuiButton--caution">Caution</button>

States

Disabled

Use when another action has to be completed first, such as changing a value in a field or waiting for a system response. It is used only with default and primary buttons, and it is normally dimmed until it is enabled.

<button class="tuiButton" disabled="disabled">Disabled Default</button>
<button class="tuiButton tuiButton--primary" disabled="disabled">Disabled Primary</button>
<button class="tuiButton tuiButton--subtle" disabled="disabled">Disabled Subtle</button>
<button class="tuiButton tuiButton--caution" disabled="disabled">Disabled Caution</button>

Active

Indecates the button is pressed.

<button class="tuiButton tuiButton-is-active">Default Active</button>
<button class="tuiButton tuiButton--primary tuiButton-is-active">Primary Active</button>
<button class="tuiButton tuiButton--subtle tuiButton-is-active">Subtle Active</button>
<button class="tuiButton tuiButton--caution tuiButton-is-active">Caution Active</button>

Semantics

Anchors

You can use anchors as underlying elements.

Default Primary Subtle Caution 
<a class="tuiButton" href="#">Default</a>
<a class="tuiButton tuiButton--primary" href="#">Primary</a>
<a class="tuiButton tuiButton--subtle" href="#">Subtle</a>
<a class="tuiButton tuiButton--caution" href="#">Caution</a>

Inputs

You can use inputs as underlying elements. For buttons, available types are: button, submit and reset.

<input type="button" class="tuiButton" value="Button" />
<input type="submit" class="tuiButton tuiButton--primary" value="Submit" />
<input type="reset" class="tuiButton tuiButton--caution" value="Reset" />

Guidelines

Labels

Make sure the purpose of the button is clear to the user. Use concise text that clearly describes the action the button performs. Labels are sentence case and are often a single word, a verb.

Order

Buttons are right-aligned on the screen. Place the affirmative or expected action on the left, the dismissive action on the right. The exception to the rule is the "Back" button, which would be on the left.
