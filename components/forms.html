Forms Draft

Use forms to allow people to enter data for use by the application, or to configure options. Forms are used to get information and guide people with minimal fuss. Designing forms well requires making decisions about structure, sequence, interface elements, field labels, help, and feedback.

Form background color

The gray background color applied to the forms in the following examples is only to show form elements box boundaries.

Default form layouts

A form tuiForm is composed of tuiForm__row elements. Each is composed of at least one column tuiForm__col. The column hosting the field label must apply the tuiForm__col--label modifier.

There's two row modifiers tuiForm__row--checkbox and tuiForm__row--no-input availble to help align checkbox and non-input fields label.

Labels
Standard label alignment is left-aligned with the field to its right.
Required field label
Mark required fields with an asterisk (*). If all fields in the form are mandatory, this isn't necessary.
Disabled fields
Show disabled fields if users need to know what controls might be available to them.
Field length
The length of the field should reflect the intended length of content.

Footer

Form actions should be added within a tuiForm__Footer element. All elements added to the tuiForm__footer__actions will be aligned to the right. Use a primary button for the main action, and a subtle button to cancel. 

<form class="tuiForm">
   <div class="tuiForm__row">
       <div class="tuiForm__col tuiForm__col--label">
           <label class="tuiLabel" for="ffF1">Name</label>
       </div>
       <div class="tuiForm__col">
           <input class="tuiTextField" type="text" id="ffF1" />
       </div>
   </div>
   <footer class="tuiForm__footer">
       <div class="tuiForm__footer__actions">
           <button class="tuiButton tuiButton--primary">Submit</button>
           <button class="tuiButton tuiButton--subtle">Cancel</button>
       </div>
   </footer>
</form>
