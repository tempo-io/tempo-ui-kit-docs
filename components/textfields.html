Text Fields Draft

Use text fields to allow users to enter, select and search for text.

Single line

Default

<input class="tuiTextField" type="text" />
<input class="tuiTextField" type="text" value="With some text..." />

Place holder

Placeholder text provides users with a short hint or example that describes the expected value for the text field. For example, 'Start typing to see names', or shows an example of a typical entry such as 'My first project'. You must prefix examples with 'e.g.' Only provide placeholder text where clarification is essential. In most cases, the label will provide sufficient guidance.

<input class="tuiTextField" type="text" placeholder="Placeholder text" />

Disabled

Use disabled fields when there is a prerequisite condition that must be met before the user can enter a value, or when there is a scenario where that field would become available.

<input class="tuiTextField" type="text" disabled="disabled" />
<input class="tuiTextField" type="text" placeholder="Placeholder text" disabled="disabled" />
<input class="tuiTextField" type="text" value="With some text..." disabled="disabled" />

Size

Different modifiers are available to set the input width. Use tuiTextField--date for ISO date inputs and tuiTextField--hours for time entry.

<input class="tuiTextField tuiTextField--date" type="text" placeholder="2017-01-01" />
<input class="tuiTextField tuiTextField--hours" type="text" placeholder="00:01" />

Multi-line

Default

<textarea class="tuiTextField"></textarea>
<textarea class="tuiTextField">With some text...</textarea>

Place holder

Placeholder text provides users with a short hint or example that describes the expected value for the text field. For example, 'Start typing to see names', or shows an example of a typical entry such as 'My first project'. You must prefix examples with 'e.g.' Only provide placeholder text where clarification is essential. In most cases, the label will provide sufficient guidance.

<textarea class="tuiTextField" placeholder="Placeholder text"></textarea>

Disabled

Use disabled fields when there is a prerequisite condition that must be met before the user can enter a value, or when there is a scenario where that field would become available.

<textarea class="tuiTextField" disabled="disabled"></textarea>
<textarea class="tuiTextField" placeholder="Placeholder text" disabled="disabled"></textarea>
<textarea class="tuiTextField" disabled="disabled">With some text...</textarea>

Resizability

Multi-line text fields must remain resizable at all times. You can restrict It to resize vertically only, but there should always be a way to make the text field larger.

<textarea class="tuiTextField">Default resize</textarea>
<textarea class="tuiTextField tuiTextField--vertical-only">Vertical only resize</textarea>

Guidelines

Length

The length of the text field should reflect the expected length of content. This guideline applies to any form controls with a similar shape, such as drop-down menus and text boxes. Keep localization in mind when determining the length of your fields.

Accessibility

Auto-focus the first field by default. Users should then be able to tab through elements in the form in a logical sequence. This maintains Section 508 accessibility compliance.
